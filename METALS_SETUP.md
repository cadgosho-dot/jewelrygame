# 地金相場の自動更新・初回設定

この版には、Metals.Dev → GitHub Actions → `data/metals.json` → JEWELRY×JEWELRY地金屋、という自動更新機能が入っています。

## 毎日の動作

- `main`へゲームをPushした直後に相場を再取得してから公開
- 平日・日本時間 9:43 に自動更新
- 平日・日本時間 14:17 に自動更新
- 土日にはAPIを呼ばず、金曜日の最終価格を使用
- 川原さんのパソコンや携帯が起動していなくてもGitHub上で動作
- 取得失敗時はGitHub Pagesの公開を中止し、直前の正常な公開版を維持

## 最初に一度だけ行うこと

1. Metals.Devで無料アカウントを作成し、APIキーを取得します。
2. APIキーをチャットやゲームファイルへ貼らないでください。
3. GitHubでJEWELRY×JEWELRYのリポジトリを開きます。
4. `Settings` → `Secrets and variables` → `Actions` を開きます。
5. `New repository secret` を押します。
6. Nameへ `METALS_DEV_API_KEY` と入力します。
7. SecretへMetals.DevのAPIキーを入力して保存します。
8. GitHubの `Settings` → `Pages` を開きます。
9. `Build and deployment` の `Source` を **GitHub Actions** に変更します。
10. GitHubの `Actions` タブを開きます。
11. 左側の「地金相場更新とGitHub Pages公開」を開き、`Run workflow` を一度実行します。
12. 実行が成功し、`data/metals.json` が自動コミットされ、GitHub Pagesも公開されたことを確認します。

## GitHub側で書き込みが拒否された場合

`Settings` → `Actions` → `General` → `Workflow permissions` で、ワークフローがリポジトリ内容を書き換えられる設定を確認してください。組織やブランチ保護の設定によっては、管理者側の許可が必要です。

GitHub Actionsの`GITHUB_TOKEN`で作成したコミットは、従来の「Deploy from a branch」方式のPagesビルドを自動起動しません。そのため、この版は同じワークフロー内で価格更新後にGitHub Pagesへ直接公開します。PagesのSourceは必ず「GitHub Actions」にしてください。通常のゲーム更新を`main`へPushしたときも、同じワークフローが自動公開します。

v0.10.139以降は、通常のゲーム更新を`main`へPushした場合も、先にPush前の正常な相場・前日比・履歴を引き継ぎ、その後Metals.Devから相場を再取得します。取得結果が正常な`live`状態であることを検証してから公開するため、完全入れ替え用ZIPに入っている初期`fallback`データがそのまま公開されることを防ぎます。

## ゲーム内の計算

- K18購入価格：金の1g相場 × 0.75 × 1.15
- K18売却価格：金の1g相場 × 0.75 × 0.98
- Pt900購入価格：プラチナの1g相場 × 0.90 × 1.15
- Pt900売却価格：プラチナの1g相場 × 0.90 × 0.94
- SV925購入価格：銀の1g相場 × 0.925 × 1.30
- SV925売却価格：銀の1g相場 × 0.925 × 0.90
- K18・Pt900は10円単位、SV925は1円単位で四捨五入
- 地金屋では1g単位で購入・売却

上乗せには、ゲーム内地金屋の小口販売、割金、加工・流通分を含めています。


## 純金属相場と前日比

- 地金屋を開くと、純プラチナ・純金・純銀のJPY／g相場を表示します。
- 日本時間の日付が変わった最初の更新時に、直前に保存されていた正常相場を前日比較用として保持します。
- 同じ日の9:43と14:17の更新では、比較対象の前日相場は変えません。
- 土日や休場日を挟む場合は、直近に正常更新された日の相場と比較します。
- 上昇は緑の「＋〇〇円」、下落は赤の「−〇〇円」、同値は「±0円」と表示します。
- 初回更新直後など比較元がない場合は「前日比 ─」と表示し、翌更新日から差額を表示します。

## 無料枠を守る設計

平日1日2回だけ取得するため、通常は月40〜46回程度です。プレイヤー数が増えても、各プレイヤーはGitHub Pages上の同じ `metals.json` を読むだけなので、Metals.Devへの取得回数は増えません。

## 異常時

- APIキー未設定：ワークフローは失敗し、既存価格は維持
- API障害：既存価格は維持
- 異常値：更新を中止し、既存価格は維持
- ゲーム側でJSONを取得できない：ゲーム内基準価格を使用
- 72時間以上更新されていない：地金屋に「前回取得した地金相場」と表示

## ゲーム内の売買単位

- K18、Pt900、SV925は1g単位で購入・売却します。
- プラチナは通常Pt900のみを扱います。
- 地金屋では「購入価格」と「売却価格」を別々に表示します。
- K18：購入＝金相場×0.75×1.15、売却＝金相場×0.75×0.98。
- Pt900：購入＝プラチナ相場×0.90×1.15、売却＝プラチナ相場×0.90×0.94。
- SV925：購入＝銀相場×0.925×1.30、売却＝銀相場×0.925×0.90。
- K18・Pt900は10円単位、SV925は1円単位で四捨五入します。

## 相場履歴グラフ

- GitHub Actionsが正常更新するたびに、`data/metals.json`の`historyDaily`へ実際の日付と純金属相場を保存します。
- 同じ日の9:43と14:17では、その日の記録を最新価格へ上書きします。
- 履歴は自動的に最大5年間だけ保持します。
- 月間グラフは日ごとの記録を表示します。土日・休場日・取得失敗日など、記録がない日は点を表示しません。
- 年間グラフは、記録された日次価格から各月の平均価格を計算して表示します。
- 実装前の過去相場を外部から遡って取得する機能はありません。履歴は、この版で自動更新を開始した日から蓄積されます。
- ゴールドはオレンジ、プラチナは銀色、シルバーは青で表示します。
