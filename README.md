## GitHub Desktop 反映方法
1. 既存リポジトリの中身をいったん全削除します（`.git` フォルダは消さない）。
2. この ZIP の中身をそのままコピーします。
3. GitHub Desktop で変更を確認します。
4. Commit → Push origin を実行します。

公開URL：`https://cadgosho-dot.github.io/jewelrygame/`

## v0.5.8 店舗契約と接客条件
- ゲーム開始時点では店舗を持っていません。
- 不動産で小さな店舗を契約すると、店舗メニューへ切り替わります。
- ジュエリーを1点以上制作した後から、お客様が来店して接客できるようになります。


## v0.5.9 御徒町と店舗
- 不動産屋は御徒町内にあります。
- メインメニューの店舗ボタンは、店舗未契約時には押せません。
- 御徒町の不動産屋で店舗を借りると、店舗ボタンが利用可能になります。


## v0.6.0 原石とルース
- 採掘：原石のみ入手
- 素材屋：地金・ルースの購入、原石の売却
- g-Lab.：原石研磨機の購入（初期価格30,000円）
- 工房：研磨機がある場合、原石1個を2時間で同種のルース1個へ加工
- ジュエリー制作：ルースと地金を使用


## v0.6.2 中央配置UI
- メイン画面以外の操作UIを中央へ配置しました。
- 操作枠とボタンは背景を塗らず、枠線と文字だけで表示します。


## v0.6.4 名前設定
- 初回開始時に名前を入力します。
- 名前はゲーム内の設定から後で変更できます。
- 主人公画像やキャラクター作成機能はありません。


## v0.6.5 店舗名
- 店舗を借りる際に、プレイヤーが店舗名を決定します。
- 最初の店舗は本店として登録されます。
- 将来の支店は2号店、3号店のように区別できる構成です。


## v0.6.6 名前の利用
- 設定画面からプレイヤー名と店舗名を変更できます。
- 店舗名は全支店で共通です。支店区分は本店、2号店、3号店のように別管理します。
- プレイヤー名と店舗名は、会話や通知の中で必要な場面だけ自然に表示します。


## v0.6.7 g-Lab.の配置
- g-Lab.はメインメニューではなく、御徒町内から利用します。
- 御徒町では「素材屋」「不動産屋」「g-Lab.」を選択できます。


## v0.6.9 工房メニュー
- 工房の操作は「原石研磨」と「ジュエリー作成」の2項目です。
- 原石研磨には、御徒町のg-Lab.で購入できる研磨機が必要です。


## v0.7.0 店舗・御徒町メニュー
- 店舗画面は店舗名と本店・2号店・3号店の一覧から選択する構成です。
- 店舗がない場合は「現在店舗はありません」と表示します。
- 御徒町には8施設を表示し、初期状態では素材屋とg-Lab.だけ利用できます。
- ルースの購入は素材屋ではなくルース屋で行います。
